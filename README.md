# korfix-assistant

AI-ассистент для работы с данными Korfix ERP через Claude Code. Отвечает на бизнес-вопросы, строит сводки, ищет записи, создаёт задачи — через MCP без написания кода.

В отличие от [korfix-devkit](https://github.com/korfixdev/devkit), который для **разработки миниапов** (код, zip, деплой), этот плагин для **использования платформы** — руководителей, менеджеров, автоматизаций.

---

## Установка

```
/plugin add github:korfixdev/assistant
```

## Настройка окружения

Плагин работает **только через MCP** — без токена и MCP-URL он не функционален.

```bash
export KORFIX_TOKEN="your-token-from-db-api"
export KORFIX_MCP_URL="https://mcp.korfix.ru/${KORFIX_TOKEN}/sse"
```

Получить токен:
1. В панели Korfix → `/db/api` → Добавить
2. Выдай токен с доступом к нужным каталогам (чем больше — тем больше может ассистент)
3. Экспортируй в env

Для self-hosted Korfix — свой MCP-endpoint.

## Что внутри

### Агент

| Агент | Роль |
|-------|------|
| `korfix-ops` | Отвечает на бизнес-вопросы пользователя: сводки, поиск, создание записей через MCP |

### Skills

| Skill | Когда |
|-------|------|
| `korfix-data-read` | Чтение записей из каталогов (список, одна запись, фильтры) |
| `korfix-find-records` | Поиск по условиям: задачи, клиенты, сделки, проекты |
| `korfix-data-modify` | Создание/обновление записей (задачи, клиенты, заметки) |
| `korfix-catalog-reference` | Навигатор по каталогам платформы — где какие данные лежат |

### Документация

- `docs/catalogs.md` — полный список каталогов Korfix ERP (финансы, задачи, продажи, склад и т.д.)

Полная документация платформы — [docs.korfix.ru](https://docs.korfix.ru) и [korfixdev/docs](https://github.com/korfixdev/docs).

## Примеры использования

После установки и настройки env:

```
Покажи мне все задачи со статусом "В работе" за последнюю неделю
```

```
Сколько сделок закрыто в марте? Сгруппируй по менеджерам
```

```
Создай задачу "Позвонить клиенту ACME" на Алексея Григорьева, срок — завтра
```

```
Найди клиента с именем похожим на "Петров" — покажи контакты и последние сделки
```

Агент:
1. Проверит доступ к нужным каталогам через токен
2. Сделает запросы через MCP (`db_read`, `catalog_schema`, `db_insert`, `db_update`)
3. Вернёт результат в читаемом формате

**Агент не пишет код миниапов.** Для этого — `korfix-devkit`.

## Безопасность

- Токен = уровень доступа ассистента. Выдавай **минимум прав**: только каталоги, которые можно читать/править этим юзером.
- Для автоматизаций и скриптов — отдельный токен (не личный), с ограниченным scope.
- Ассистент может создавать/менять записи — если токен это разрешает. Будь осторожен с правами на запись.

## Когда какой плагин ставить

| Задача | Плагин |
|---|---|
| Написать новый миниап для маркетплейса | `devkit` |
| Задеплоить обновление миниапа | `devkit` |
| Получить сводку «сколько задач у команды» | `assistant` |
| Построить финансовый отчёт из данных | `assistant` |
| Создать клиента/задачу через команду голосом/текстом | `assistant` |
| Автоматизация через n8n / внешний скрипт | только MCP, плагин не нужен |

Можно поставить **оба плагина одновременно** — они не конфликтуют.
