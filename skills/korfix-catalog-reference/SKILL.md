# korfix-catalog-reference

Маппинг «что пользователь имел в виду» → «какой каталог использовать».

## Как работать с этим skill

Пользователь говорит на бизнес-языке («задачи», «сделки», «финансы»), а MCP работает с каталогами (`tt_tasks`, `ag_sales`, `ag_cashflows`). Здесь — перевод.

Полный справочник каталогов с описаниями — `${CLAUDE_PLUGIN_ROOT}/docs/catalogs.md`. Читай его при первом знакомстве с системой или когда не уверен.

## Мапинг запросов

| Пользователь говорит | Каталоги |
|---|---|
| «задачи», «taski», «todo» | `tt_tasks` |
| «клиенты» | `ag_clients` (юрлица) + `ag_persons` (физлица) — уточни какие |
| «сделки», «продажи» | `ag_sales` |
| «деньги», «движение средств», «приход/расход» | `ag_cashflows` |
| «проекты» | `ag_projects` (бизнес) или `tt_projects` (задачные) — уточни |
| «склад», «остатки» | `wh_stocks`, `wh_products` |
| «заказы» | `b2b_orders` (торговля), `md_orders` (производство) — уточни |
| «сотрудники», «команда» | `auth_pers` |
| «БДР», «бюджет» | `ag_cashflows` + агрегация (см. `korfix-finance-reports` если есть) |

## Неочевидные каталоги

Если пользователь спрашивает про что-то, чего нет в шпаргалке:
1. Прочитай `${CLAUDE_PLUGIN_ROOT}/docs/catalogs.md`
2. Если и там нет — через `catalog_schema` попробуй по предполагаемому префиксу (`custom_X`, `crm_X`)
3. Уточни у пользователя: «У вас в системе может быть кастомный каталог — как он называется в навигации?»

## Кастомные каталоги

Все каталоги с префиксом `custom_` — созданы через self-provisioning из миниапов. Список:

```
db_read('custom_dbtables', { limit: 999 })
```

Это покажет alias каждого кастомного каталога и что они хранят.

## Схема перед работой

При первом обращении к незнакомому каталогу:
```
catalog_schema('ag_cashflows')
```

Узнаешь: поля, типы, обязательность, FK-связи, enum-значения.
